# Project Charter（v0.1 / WIP）

## 1. プロジェクト目的（Why）
契約ルール判定（解約・停止・返金保証など）に関する判断業務を整理・可視化し、属人化を防ぎ、判断根拠を共有可能な状態にする。

## 2. 対象ユーザー / 利用者（Who）
- CSオペレーター（一次対応）
- 業務管理者 / SV（判断基準の管理・確認）

## 3. 成果物（What）
- 契約ルール判定に関する業務フロー定義
- 判断に使用されるナレッジの構造化定義（Knowledge Unit）
- ナレッジを検索・参照できるMVP（プロトタイプ）
- 仕様変更および判断変更の履歴ドキュメント

## 4. スコープ
### In Scope
- 解約・停止・返金保証に関する業務判断

### Out of Scope
- 決済・請求処理の変更
- 顧客向け自動応答
- AIによる最終判断の自動化

## 5. 前提・制約（Constraints）
### 技術的前提
- 初期は既存媒体（CSV / Google Sheets / 簡易UI）を前提
- MVP段階は精度より構造を優先

### 業務的前提
- 契約内容は商品・コースごとに異なる
- ルールは将来的に変更される

### 非機能要件
- 後任者がドキュメントだけで全体像を理解できる
- 判断理由と変更履歴が追跡可能

## 6. 成功条件（Doneの定義）
- 現行業務フローと判断基準が第三者に説明できる
- 判断根拠となるナレッジを検索・参照できる
- なぜその判断に至ったかを言語化できる
