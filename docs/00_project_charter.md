# Project Charter（WIP）

## 1. プロジェクト目的（Why）
本プロジェクトは、コールセンター業務における
契約ルール判定（解約・停止・返金保証など）に関する判断業務を整理・可視化し、
属人化を防ぎ、判断根拠を共有可能な状態にすることを目的とする。


## 2. 対象ユーザー / 利用者（Who）
主な利用者：
- CSオペレーター（一次対応）
- 業務管理者 / SV（判断基準の管理・確認）

想定しない利用者：
- 顧客向け自動応答システム
- 完全自動での契約判断を行うシステム


## 3. 成果物（What）
- 契約ルール判定に関する業務フロー定義
- 判断に使用されるナレッジの構造化定義
- ナレッジを検索・参照できるMVPシステム
- 仕様変更および判断変更の履歴ドキュメント
 

## 4. スコープ
### In Scope
- 解約、停止、返金保証に関する判断業務
- 判断根拠となる契約条件・社内ルールの整理
- 例外対応の記録と参照


### Out of Scope
- 決済処理、請求処理そのものの変更
- 顧客向けFAQ自動回答
- AIによる最終判断の自動化


## 5. 前提・制約（Constraints）
- 技術的前提：
    - 初期段階では既存ツール（CSV / Google Sheets / 簡易UI）を前提とする
    - MVP段階では精度よりも構造を重視する
- 業務的前提：
    - 契約内容は商品・コースごとに異なる
    - 判断基準は将来的に変更される可能性がある
- 非機能要件：
    - 後任者がドキュメントを読めば全体像を理解できること
    - 判断理由と変更履歴が追跡可能であること

## 6. 成功条件（Doneの定義）
- 現行の契約判断業務フローを第三者に説明できる
- 判断根拠となるナレッジを検索・参照できる
- なぜその判断に至ったかを言語化できる
