# KU-02：縛り回数のカウント定義（v0.1）

## 定義
- 縛り回数判定は「発送済み受注の数」をカウントする
- 返品受注はカウントに含めない
- “7回目”は「7回目まで発送済み」を指す（＝縛り達成）

## 信頼できないデータ（注意）
- ECForceCS画面 定期受注管理の「定期お約束回数」は設定ミスが発生するため信用しない

## 事故ポイント（よくある誤認）
- 返品受注があるにも関わらず、画面上の“発送回数”が7等に見えて縛り達成と誤認する
- 商材一覧/オーダー変更により解約条件の解釈を誤る（アップセル前の受注数の扱い等）

## 現場対応の回避策（暫定）
- 「受注一覧の発送済み受注」を自前で数える（返品除外）
- “定期お約束回数”は参照値として扱い、判断根拠にしない
