# Decision Log（v0.1）

## 2025-12-23: 縛り回数カウントのソース・オブ・トゥルース
- 決定：縛り回数は「発送済み受注数」で判定し、返品受注は除外する
- 理由：画面上の“発送回数”は返品混入で誤認が起き、事故が過去に発生しているため

## 2025-12-23: 信頼できない項目の明示
- 決定：ECForce定期受注管理の「定期お約束回数」は設定ミスが多いため信用しない
- 理由：運用上、設定ミスが度々発生しており判断根拠にできない

## 2025-12-23: コース名の正規化
- 決定：旧「7回受け取りお約束コース」は新「7回お約束コースLN」として扱う
- 理由：現場運用・表記統一のため（旧称は注記として保持）

## 2025-12-23: アップセル時の縛り回数継承（漲天）
- 決定：漲天パワフル → 7回お約束コースLN の場合、アップセル前の発送済み受注を縛り回数に含める
- 根拠：スプレッドシート + 議事録 + Chatwork の複合情報
- 理由：含めないと誤処理（縛り未達扱い）になりやすく、解釈違い事故が発生しうるため

## 2025-12-23: ルール改定（時期で変わる）の現状運用
- 事実：覚えている人に聞き、Chatwork/議事録検索で根拠を掘り起こし展開している
- 含意：ルールバージョン管理が属人化しており、知識蓄積が難しい主要因になっている
