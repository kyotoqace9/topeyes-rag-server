# 01_current_operation_flow.md (v0.1)

## 【問い合わせ概要】
解約希望

## 【前提条件】
・商品名：漲天
・コース名：7回受け取りお約束コース
・契約回数 / 利用状況：7回目
・問い合わせタイミング：次回発送予定日の16日前
・漲天パワフルコースから7回受け取りお約束コースにアップセル
・7回受け取りお約束コースは7回の受け取り前は原則解約不可



## 【オペレーターの行動フロー】
# お問い合わせ内容の確認 
  -[顧客管理>お問い合わせ管理画面]で、【解約について】【未対応】を選択して検索する
  -対応する問い合わせの【詳細】を新しいタブで開く
  -お問い合わせ内容を確認する
    -お問い合わせ内容が解約希望のみであることを確認する
# お客様情報の確認
　-【顧客番号】を新しいタブで開く([顧客管理>顧客管理画面]へ遷移)
  -顧客番号の隣の【CS画面へ】を開く([顧客管理>CS画面]へ遷移)
  -開いたCS画面で[受注一覧][定期受注一覧][顧客メモ]の3点を確認し、お問い合わせ内容に対する回答を判断する
   -[受注一覧]のコース名・対応状況・発送回数・発送予定日を確認する
     - 発送回数をみたしていること
     - 別フロー対応中でないこと(返品あり・返金保証・クーリングオフ・解約金請求中など)
     - 未発送受注のありなし
      - 未発送があれば三発送分の発送予定日(解約期限内であること)
     - コースの契約条件や注意点について商品情報スプレッドシートやクライアントからの指示(Chatwork、会議、マニュアル、エスカレーションシート等)を確認する
   -[定期受注一覧]をのコース名・ステータス・次回発送予定日を確認する
    - コース名(コースが切り替わっていないか)
    - 次回発送予定日(解約期限内であること)
    - ステータスが有効であることを確認
     - 解約済ではないか、他のフローに乗っていないか
   -[顧客メモ]の内容を確認する
     - 別フローに乗っていないか
# お問い合わせを対応済みに変更する
  -[お問い合わせ履歴]の対応するお問い合わせの【編集】を開く
  -ステータスを対応済みに変更して保存する
  -ステータスが対応済みになる
# 解約処理
  -[定期受注一覧]の解約したい定期受注の【編集】を開く
  -ステータスをキャンセルに変更する
  -そのまま最下部までスクロール(Endキー)して保存を押下した後、画面を更新する(F5キー)
  -[定期受注]のステータスがキャンセルに変更される
# 解約受付処理完了メールを送信する
  -[受注一覧]で解約するコースの最終受注にチェックを付け、メール一括通知を開く※最終受注の発送予定日を確認しておく
  -[テンプレートから選択する]プルダウンメニューの「【解約】メール解約受付」テンプレートを選択する
  -「【解約】メール解約受付」が選択されていることを確認し、【プレビュー】を開く
  -メールプレビュー本文内の最終商品の発送予定日が誤った日付になっていないか確認し、メールプレビューを閉じる
  -[テンプレートから選択する]部の【この内容で送信する】ボタンを押下し、メールを送信する ※[本文を入力する] 側のボタンは使用しない
# 顧客メモ登録
  -顧客メモの内容欄に自分の名前を追加したテンプレートを張り付ける
	-【ECFお問い合わせ】
	-受信日時：
	-・解約希望（例文）
	-→メール解約受付処理
	-担当：会社名+自身の名前
  -メモ内容の[受信日時]は[お問い合わせ履歴]等からコピーする
  -[顧客メモ]の内容欄に受信日時を追記して保存する
  -[顧客メモ]が反映される
# 最終チェック
  -[顧客メモ]に漏れ・誤りがないか確認する
  -[定期受注一覧]で対応に誤りがないか確認する
  -[受注一覧]で対応に誤りがないか確認する
  -メールが送信されたことを確認する

## 【判断ポイント】
- どこでYes / Noが分かれるか
     - 発送回数
     - 次回発送予定日
     - 別フロー対応中
     - 特定ワード提示
     - 温度感上昇
- 条件分岐が発生する箇所
     - お問い合わせ内容の確認 
     - [受注一覧]のコース名・対応状況・発送回数・発送予定日を確認する
     - [定期受注一覧]をのコース名・ステータス・次回発送予定日を確認する
     - [顧客メモ]の内容を確認する


## 【迷いやすい点】
  - 温度感が高めかどうか
  - 発送回数は何回か
  - 縛り達成はどう数えたらいいか
  - 顧客メモの内容で気にしなければいけないのはどれか
  - ルールを確認するにはどの資料を参照したらいいか
  - 特殊な対応にしなければならないのに見落としていないか
  - どこを確認すれば問題なく対応できるか

## 【例外・過去トラブル】
  - 縛り回数達成しているはずなのに発送カウントを間違えており多く送ってしまって着払い返品案件になった
  - 縛り回数達成の設定を間違えており、達成しても自動解約でokがでず、着払い返品になった
